1.单页面应用程序

1.1 网页交互方式

  • 经典多页面
    • 前后端柔杂在一起，不利于维护
    • 用户体验一般，点击跳转，等待时间过长
    • 每个页面都要重新渲染
    • 有利于SEO搜索引擎搜索到
  • 现代式单页面
    • 前后端分离，开发效率高，可维护性好
      • 服务端只关心数据
      • 客户端只关心页面，通过接口拿数据，处理页面
    • 用户体验好，不需要所有都重新刷新
    • 只需要渲染局部视图
    • 单页面开发技术复杂，所以开发了许多框架
      • AngularJS
      • ReactJS
      • VueJS
    • 单页面技术已经很成熟了，但无法顾及到低版本浏览器
    • 现在除了电商网站，很少有网站兼容低版本浏览器
    • 单页面数据是异步加载过来的，所以不利于SEO搜索

1.2多页面：以服务端为主，前后端混合

  • 不利于开发与维护

1.3 单页面：前后端各司其职

  • 服务端只关心数据
  • 客户端只关心页面（二者通过接口来交互）

1.4模拟前后端分离开发模式

  • 项目立项
  • 需求分析
  • 服务端工作
    • 需求分析
    • 设计数据库
    • 设计接口（有时候也需要前端参与其中）
    • 接口（处理数据）开发
  • 前端工作
    • 需求分析
    • 写页面
    • 写功能
    • 通过接口和服务端进行交互

2.前端三大框架

单页开发其实是比较复杂的，需要有一些技术的支撑

所以有了前端三大框架

  • angular
    • 09年诞生
    • Google
    • 他的目的就是让我们开发单页应用程序更方便了
    • 但是他最主要是为前端带来了MVVM 开发模式
    • MVVM：数据驱动视图，不操作 DOM
  • react
    • Facebook 公司自己开发了一个Web 框架
    • 组件化
  • vue
    • 作者：尤雨溪
    • 早期由个人开发

7.组件

7.1组件起步

7.1.1全局组件与局部组件

  • 通用组件设置为全局，如轮播图以及一些公共不涉及业务的组件
  • 涉及业务的一般定义为局部组件

7.2组件注意

  • 组件 有自己独立的作用域
    • 外部访问不到内部，内部也访问不到外部
    • 就像 Nodejs 中的 js 模块一样
  • 组件其实就是一个特殊的 Vue 实例，他有自己的 data，methods，computed，watch
  • 组件的 data 必须是一个函数，返回的对象作为 data

7.3组件之间的通信：Props Down

  • 父子组件通信 Props Down

    在父组件当中通过子组件标签属性传递数据

7.3.1单向数据流

  • prop 是单向绑定的：当父组件的属性变化时，会传导给子组件，但是反过来不会

注意！！！

引用类型可以修改，但是不要这么做，因为这样就违背了 Vue 的组件通信原则

7.4组件间通信：Props up

  • 第一步：在父组件中定义一个方法（处理子组件传过来的数据的方法）
  • 第二部：在子组件内部调用该方法
    • 在子组件中发布一个自定义事件，告诉父组件你可以去添加任务了
  • 在父组件使用子组件的标签上去订阅子组件内部发布的事件
