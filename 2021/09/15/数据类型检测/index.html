你真的了解 JS 中的数据类型检测吗？

发表于 2021-09-15 更新于 2021-09-26 分类于 技术

js中的数据类型分为两大类~

基本类型：Number、String、Boolean、Null、Undefined、Symbol
复杂类型：Object、Function、Array

一、通过typeof检测数据类型

我们通过浏览器控制台来看一下用typeof检测数据类型的状况

   typeof null 居然是 object类型的
   原因：在计算机底层，数据类型的值是以二进制数储存的
         typeof是基于这个二进制值进行类型判断的
         恰巧 object 和 null 都是以 000 开头的，所以检测 null 为 object类型
         这是计算机的一个bug

typeof总结：
(1)能够完美的检测出基本类型以及function函数类型（null除外，底层bug导致）。
(2)其他的复杂类型都归属object，所以我们无法判断出归属于object类型的这些到底属于哪一种类型

二、通过instanceof检测数据类型

typeof能够完美检测基本类型，所以我们使用 instanceof 主要研究是否能检测复杂类型

  instanceof 主要检测当前实例是否属于这个类
  console.log(1 instanceof Number); // false (不能检测基本类型)
  
  let arr = []; // 创建一个数组
  console.log(arr instanceof Array);  // true
  console.log(arr instanceof Function);  // false
  console.log(arr instanceof Object);  // true

我们发现 arr 不管是 Array类型 还是 Object类型，都检测为 true

原因：只要当前类出现在实例的原型链上，结果就为true
arr的原型链包含 Array、Object 。所以，不管是 Array类型 还是 Object类型，都检测为 true

这样虽然也能检测 arr 为 Array 类型，但还是有缺陷，看下面这个例子

function Fn(){
    this.name ='我是fn';
}
Fn.prototype = Object.create(Array.prototype);// 改变 Fn 的原型指向
let fn = new Fn();
console.log(fn instanceof Array); // true;


可以看到，fn 居然变成了 Array 类型,这是因为改变了他的原型指向。

instanceof总结：
(1)不能检测基本类型
(2)因为我们可以肆意的修改原型的指向，所以测出来的类型不准确。

三、通过 constructor 检测数据类型

可以通过一个类型的 constructor 来判断类型

   let num = 1;
      let arr = [];
      console.log(num.constructor === Number);  //true
      console.log(arr.constructor === Array);  //true
      console.log(num.constructor === Object);  //false

使用 constructor 可以检测普通类型和复杂类型，，但也有缺点，下面来看看

   Number.prototype.constructor = 'ABC',
      console.log(num.constructor);  // 'ABC'

可以看到，num 的 constructor 变成了’ABC’

constructor总结：
(1)可以检测基本类型和复杂类型
(2)可以随意修改原型上的constructor，所以测出来的类型不一定准确

四、通过 Object.prototype.toString.call([value]) 检测数据类型

标准检测数据类型的方法：Object.prototype.toString 不是转换为字符串，是返回当前实例所属类的信息,在 Object 下的 toString 方法。

   let obj = {};
      console.log(obj.toString());  // [object Object]

只要调用 Object 上的 toString 方法就能返回 [object 实例的类型] 这种格式的类型

猜测：是不是只要把Object.prototype.toString 执行，让他里面的 this 变为要检测的值，那就能返回当前值所属类的信息

很明显，我们的猜测是对的，Object.prototype.toString.call(),可以检测出所有类型。这种方法虽然检测的很全面，但是比较复杂。检测基本类型我们可以使用typeof，所以我们来封装一个万能的类型检测函数~

直接调用 toType(obj) 函数 ，传入要检测的数据，即可返回响应的数据类型，这样就大功告成啦~
